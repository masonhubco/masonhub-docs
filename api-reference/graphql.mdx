# GraphQL API (v2)

The MasonHub GraphQL API provides a modern, flexible interface for querying and mutating data in the Order Management System. It offers powerful features like nested queries, field selection, and type safety.

<Note>
The GraphQL API is available at `/api/v2/graphql` and provides access to orders, SKUs, inbound shipments, RMAs, accounts, and more.
</Note>

## Endpoint

**Base URL**: `https://app.masonhub.co/api/v2/graphql`

**Sandbox URL**: `https://sandbox.masonhub.co/api/v2/graphql`

## Authentication

All GraphQL requests require JWT Bearer token authentication in the Authorization header:

```http
Authorization: Bearer your_jwt_token_here
Content-Type: application/json
```

## Basic Query Structure

All GraphQL requests use POST method with a JSON body:

```graphql
{
  "query": "query { orders(ids: [\"order-id\"]) { id customerIdentifier status } }",
  "variables": {}
}
```

## Advanced Queries

### Nested Data Fetching

GraphQL excels at fetching related data in a single request:

```graphql
query CompleteOrderView($orderId: ID!) {
  order(id: $orderId) {
    id
    customerIdentifier
    status

    # Get account details
    account {
      name
      slug
      defaultRoutingPolicy
    }

    # Get line items with full SKU details
    lineItems {
      quantity
      price
      sku {
        customerIdentifier
        productName
        brandName
        dimWeight
        dimLength
        dimWidth
        dimHeight
      }
    }

    # Get fulfillments with nested line items
    orderFulfillments {
      status
      shipmentDueAt
      inventoryLocationId
      orderFulfillmentLineItems {
        quantity
        allocatedQuantity
        sku {
          customerIdentifier
          productName
        }
      }
    }

    # Get shipments with tracking
    shipments {
      trackingNumber
      trackingUrl
      shipmentDateTime
      shippingProvider {
        name
      }
      lineItems {
        quantity
        sku {
          customerIdentifier
        }
      }
    }

    # Timeline events
    timelineEvents {
      event
      description
      occuredAt
    }
  }
}
```

### Using Fragments

```graphql
fragment OrderSummary on Order {
  id
  customerIdentifier
  status
  priority
  submittedAt
}

fragment LineItemDetails on OrderLineItem {
  skuCustomerId
  quantity
  price
  sku {
    productName
    brandName
  }
}

query GetOrderWithFragments($orderId: ID!) {
  order(id: $orderId) {
    ...OrderSummary
    lineItems {
      ...LineItemDetails
    }
  }
}
```

## Performance Optimization

### DataLoaders

The GraphQL API uses DataLoaders to optimize database queries and prevent N+1 query problems. This means you can freely nest queries without worrying about performance degradation.

### Query Batching

Multiple queries can be batched in a single HTTP request:

```json
[
  {
    "query": "query { order(id: \"id1\") { id status } }"
  },
  {
    "query": "query { sku(id: \"id2\") { id productName } }"
  }
]
```

## Introspection

Query the GraphQL schema itself to discover available types and fields:

```graphql
query IntrospectionQuery {
  __schema {
    types {
      name
      kind
      description
    }
  }
}
```

Query specific type information:

```graphql
query TypeInfo {
  __type(name: "Order") {
    name
    fields {
      name
      type {
        name
        kind
      }
    }
  }
}
```
