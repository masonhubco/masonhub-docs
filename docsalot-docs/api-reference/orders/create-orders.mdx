---
title: 'Create Orders'
description: 'Create one or more orders with comprehensive configuration options'
api: 'POST /orders'
---

Create new orders in the MasonHub fulfillment system. Orders can be submitted individually or in batches, with support for complex routing policies, inventory allocation constraints, and custom shipping requirements.

<Note>
Orders are processed synchronously. The API returns immediately with creation status and assigned order IDs.
</Note>

## Request Body

The request body accepts an array of order objects. Each order supports the following fields:

### Required Fields

<ParamField body="customer_identifier" type="string" required>
  Unique order identifier from your system. This is your primary reference for tracking the order.
</ParamField>

<ParamField body="order_type" type="string" required>
  Order type classification (e.g., "customer", "replacement", "sample").
</ParamField>

<ParamField body="shipping_provider" type="string" required>
  Shipping carrier name (e.g., "masonhub", "UPS", "FedEx", "USPS").
</ParamField>

<ParamField body="shipper_service_level" type="string" required>
  Carrier service level. Options: `ground`, `express`, `overnight`, `two_day`.
</ParamField>

<ParamField body="submitted_at" type="string" required>
  Order submission timestamp in RFC3339 format (e.g., "2018-08-01T00:00:00Z").
</ParamField>

<ParamField body="line_items" type="array" required>
  Array of line item objects. Each item requires `sku_customer_id` and `quantity`.
</ParamField>

### Shipping Address (Required)

<ParamField body="shipping_address_name" type="string" required>
  Recipient full name.
</ParamField>

<ParamField body="shipping_address_street_line_one" type="string" required>
  Street address line 1.
</ParamField>

<ParamField body="shipping_address_street_line_two" type="string">
  Street address line 2 (apartment, suite, etc.).
</ParamField>

<ParamField body="shipping_address_city" type="string" required>
  City name.
</ParamField>

<ParamField body="shipping_address_locale" type="string" required>
  State/province code (e.g., "NJ", "CA").
</ParamField>

<ParamField body="shipping_address_postal_code" type="string" required>
  Postal/ZIP code.
</ParamField>

<ParamField body="shipping_address_country_code" type="string" required>
  Two-letter ISO country code (e.g., "US", "CA").
</ParamField>

<ParamField body="shipping_address_phone_number" type="string" required>
  Contact phone number.
</ParamField>

<ParamField body="shipping_address_type" type="string" required>
  Address type. Options: `residential` or `commercial`.
</ParamField>

### Billing Address (Required)

<ParamField body="billing_address_name" type="string" required>
  Billing contact name.
</ParamField>

<ParamField body="billing_address_street_line_one" type="string" required>
  Billing street address line 1.
</ParamField>

<ParamField body="billing_address_street_line_two" type="string">
  Billing street address line 2.
</ParamField>

<ParamField body="billing_address_city" type="string" required>
  Billing city.
</ParamField>

<ParamField body="billing_address_locale" type="string" required>
  Billing state/province code.
</ParamField>

<ParamField body="billing_address_postal_code" type="string" required>
  Billing postal/ZIP code.
</ParamField>

<ParamField body="billing_address_country_code" type="string" required>
  Billing country code.
</ParamField>

<ParamField body="billing_address_phone_number" type="string" required>
  Billing phone number.
</ParamField>

<ParamField body="billing_address_type" type="string" required>
  Billing address type: `residential` or `commercial`.
</ParamField>

### Optional Fields

<ParamField body="priority" type="integer">
  Order priority (higher values = more urgent). Range: 1-999.
</ParamField>

<ParamField body="value_added_services" type="array">
  Array of additional service strings to be performed (e.g., gift wrapping, custom packaging).
</ParamField>

<ParamField body="special_instructions" type="string">
  Special handling instructions for warehouse staff.
</ParamField>

<ParamField body="gift_message" type="string">
  Gift message to include with the order.
</ParamField>

<ParamField body="backorder_policy" type="string">
  How to handle out-of-stock items. Options:
  - `cancel_shorts` - Cancel unavailable items
  - `backorder` - Hold order until inventory available
  - `partial_ship` - Ship available items immediately
</ParamField>

<ParamField body="routing_policy" type="string">
  Fulfillment routing strategy for multi-warehouse scenarios.
</ParamField>

<ParamField body="split_policy" type="string">
  Order splitting rules. Options:
  - `single_shipment` - Never split orders
  - `allow_splits` - Allow splits for efficiency
  - `minimize_splits` - Prefer single shipments but allow splits
</ParamField>

<ParamField body="order_localization" type="object">
  Localization settings for international orders.
</ParamField>

### Line Items

Each line item in the `line_items` array supports:

<ParamField body="sku_customer_id" type="string" required>
  Your SKU identifier (must exist in catalog).
</ParamField>

<ParamField body="quantity" type="integer" required>
  Quantity to ship.
</ParamField>

<ParamField body="promised_delivery_date" type="string">
  Promised delivery date in RFC3339 format.
</ParamField>

<ParamField body="estimated_delivery_date" type="string">
  Estimated delivery date in RFC3339 format.
</ParamField>

<ParamField body="pick_from" type="array">
  Inventory allocation constraints. Each rule specifies `match_type`, `match_value`, and `match_style`.
</ParamField>

## Advanced Features

### Pick From Constraints

Control exactly which inventory is allocated to orders using `pick_from` rules:

<Tabs>
  <Tab title="Hard Match">
    **Required Constraint**: Order will fail if inventory doesn't match criteria.

    ```json
    {
      "pick_from": [
        {
          "match_type": "purchase_order",
          "match_value": ["PO23423", "PO42322"],
          "match_style": "hard"
        }
      ]
    }
    ```

    Use for: Specific lot requirements, quality control, customer-specific inventory.
  </Tab>

  <Tab title="Soft Match">
    **Preferred Constraint**: System tries to match but will use other inventory if unavailable.

    ```json
    {
      "pick_from": [
        {
          "match_type": "lot_number",
          "match_value": ["LOT001"],
          "match_style": "soft"
        }
      ]
    }
    ```

    Use for: FEFO optimization, preferred locations, non-critical preferences.
  </Tab>
</Tabs>

### Order Splitting Policies

Configure how orders can be split across multiple shipments:

<CardGroup cols={3}>
  <Card title="Single Shipment" icon="box">
    Never split orders. Wait until all items are available to ship together.
  </Card>
  <Card title="Allow Splits" icon="boxes-stacked">
    Allow orders to split freely for maximum efficiency and speed.
  </Card>
  <Card title="Minimize Splits" icon="layer-group">
    Prefer single shipments but allow splits if necessary to avoid delays.
  </Card>
</CardGroup>

### Backorder Policies

Handle out-of-stock scenarios:

<Steps>
  <Step title="Cancel Shorts">
    Cancel unavailable items immediately. Customer receives partial order.
  </Step>
  <Step title="Backorder">
    Hold entire order until all items are available. Guarantees complete order.
  </Step>
  <Step title="Partial Ship">
    Ship available items immediately, backorder the rest. Balance of speed and completeness.
  </Step>
</Steps>

## Best Practices

<Warning>
Always provide complete shipping and billing addresses with country codes. Missing address fields will cause validation errors.
</Warning>

<CardGroup cols={2}>
  <Card title="Validate SKUs First" icon="check">
    Ensure all line item SKUs exist in your catalog before creating orders.
  </Card>
  <Card title="Set Realistic Priorities" icon="ranking-star">
    Use priority judiciously. Too many high-priority orders reduce effectiveness.
  </Card>
  <Card title="Use Callbacks" icon="bell">
    Configure orderEvent webhooks for real-time status updates instead of polling.
  </Card>
  <Card title="Batch Operations" icon="layer-group">
    Submit multiple orders in single API calls for improved performance.
  </Card>
</CardGroup>

<Tip>
Store the `system_id` (MasonHub order UUID) returned in the response for future order queries and updates.
</Tip>
