---
title: 'Update Inbound Shipments'
description: 'Update one or more existing inbound shipments (ASN)'
api: 'PUT /inbound_shipments'
---

Update existing inbound shipments with new information. This endpoint supports batch updates of multiple shipments in a single request.

<Warning>
Only shipments with `open` status can be updated. Shipments that are already on dock or being received cannot be modified.
</Warning>

<Note>
Updates require a full replacement of the shipment data. Include all fields in your request, not just the ones you want to change.
</Note>

## Request Body Schema

Send an array of complete inbound shipment objects. The schema is identical to creating shipments.

### Required Fields

<ParamField body="customer_identifier" type="string" required>
  Unique shipment identifier that matches the existing shipment you want to update.
</ParamField>

<ParamField body="customer_purchase_order_id" type="string" required>
  Your purchase order reference number for this shipment.
</ParamField>

<ParamField body="inventory_location_id" type="string" required>
  MasonHub location UUID where the shipment will be received.
</ParamField>

<ParamField body="expected_arrival_date" type="string" required>
  Expected delivery date in RFC3339 format (e.g., `2018-12-11T10:00:00Z`).
</ParamField>

<ParamField body="shipper_name" type="string" required>
  Name of the shipping company or sender.
</ParamField>

<ParamField body="line_items" type="array" required>
  Array of items expected in the shipment. Each item must include:
  - `customer_sku_id` (string, required): Your SKU identifier
  - `quantity` (integer, required): Expected quantity
</ParamField>

### Optional Fields

<ParamField body="carrier_information" type="object">
  Detailed carrier information object with fields:
  - `name` (string): Carrier company name
  - `type` (string): Carrier type (e.g., "TL", "LTL", "Parcel")
  - `contact_name` (string): Primary contact name
  - `driver_name` (string): Driver's name
  - `driver_phone` (string): Driver's phone number
</ParamField>

<ParamField body="tracking_number" type="string">
  Carrier tracking number for the shipment.
</ParamField>

<ParamField body="shipper_city" type="string">
  City where the shipment is originating from.
</ParamField>

<ParamField body="shipper_locale" type="string">
  State or locale where the shipment is originating from (e.g., "MI", "CA").
</ParamField>

<ParamField body="shipper_country" type="string">
  Country code where the shipment is originating from (e.g., "US", "CA").
</ParamField>

<ParamField body="special_instructions" type="string">
  Special handling instructions for the warehouse team.
</ParamField>

<ParamField body="comments" type="string">
  Additional comments or notes about the shipment.
</ParamField>

## Update Behavior

When you update an inbound shipment:

<Steps>
  <Step title="Status Validation">
    System checks that the shipment is in `open` status
  </Step>
  <Step title="Full Replacement">
    All fields are replaced with the new data (not a partial update)
  </Step>
  <Step title="Inventory Adjustment">
    Expected inventory is recalculated based on the new line items
  </Step>
  <Step title="Callback Triggered">
    **skuInventoryChange** callbacks are sent for affected SKUs
  </Step>
</Steps>

## Common Update Scenarios

<AccordionGroup>
  <Accordion title="Changing Expected Arrival Date">
    Update the `expected_arrival_date` field while keeping all other fields the same. Don't forget to include all required fields in the request.
  </Accordion>

  <Accordion title="Modifying Line Items">
    You can add, remove, or change quantities of line items. The system will automatically recalculate expected inventory and send appropriate callbacks.
  </Accordion>

  <Accordion title="Updating Carrier Information">
    Update tracking numbers, carrier details, or contact information as shipment details become available or change.
  </Accordion>

  <Accordion title="Adding Special Instructions">
    Add or modify special handling instructions before the shipment arrives at the facility.
  </Accordion>
</AccordionGroup>

## Best Practices

## Status Restrictions

<Warning>
Shipments can only be updated when in **open** status. Once a shipment moves to:
- **onDock**: Shipment has arrived - contact MasonHub support for changes
- **receivingStarted**: Processing has begun - updates not allowed
- **receivingComplete**: Processing complete - shipment is closed
</Warning>
