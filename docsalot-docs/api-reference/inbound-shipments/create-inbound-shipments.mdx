---
title: 'Create Inbound Shipments'
description: 'Create one or more inbound shipments (ASN) to notify MasonHub of expected deliveries'
api: 'POST /inbound_shipments'
---

Create advance ship notices (ASN) to notify MasonHub about expected inventory deliveries. This endpoint supports batch creation of up to 10 shipments per request.

<Note>
When inbound shipments are created, the system automatically updates expected inventory totals and triggers **skuInventoryChange** callbacks.
</Note>

<Warning>
All line item SKUs must exist in your catalog before creating a shipment. Non-existent SKUs will result in a validation error.
</Warning>

## Request Body Schema

Send an array of inbound shipment objects. Maximum 10 shipments per request.

### Required Fields

<ParamField body="customer_identifier" type="string" required>
  Unique shipment identifier in your system. Used to reference this shipment in future operations.
</ParamField>

<ParamField body="customer_purchase_order_id" type="string" required>
  Your purchase order reference number for this shipment.
</ParamField>

<ParamField body="inventory_location_id" type="string" required>
  MasonHub location UUID where the shipment will be received.
</ParamField>

<ParamField body="expected_arrival_date" type="string" required>
  Expected delivery date in RFC3339 format (e.g., `2018-12-11T10:00:00Z`).
</ParamField>

<ParamField body="shipper_name" type="string" required>
  Name of the shipping company or sender.
</ParamField>

<ParamField body="line_items" type="array" required>
  Array of items expected in the shipment. Each item must include:
  - `customer_sku_id` (string, required): Your SKU identifier
  - `quantity` (integer, required): Expected quantity
</ParamField>

### Optional Fields

<ParamField body="carrier_information" type="object">
  Detailed carrier information object with fields:
  - `name` (string): Carrier company name
  - `type` (string): Carrier type (e.g., "TL", "LTL", "Parcel")
  - `contact_name` (string): Primary contact name
  - `driver_name` (string): Driver's name
  - `driver_phone` (string): Driver's phone number
</ParamField>

<ParamField body="tracking_number" type="string">
  Carrier tracking number for the shipment.
</ParamField>

<ParamField body="shipper_city" type="string">
  City where the shipment is originating from.
</ParamField>

<ParamField body="shipper_locale" type="string">
  State or locale where the shipment is originating from (e.g., "MI", "CA").
</ParamField>

<ParamField body="shipper_country" type="string">
  Country code where the shipment is originating from (e.g., "US", "CA").
</ParamField>

<ParamField body="special_instructions" type="string">
  Special handling instructions for the warehouse team.
</ParamField>

<ParamField body="comments" type="string">
  Additional comments or notes about the shipment.
</ParamField>

## Inventory Integration

When you create an inbound shipment, MasonHub automatically:

<Steps>
  <Step title="Updates Expected Inventory">
    The expected inventory totals are increased for all line item SKUs
  </Step>
  <Step title="Triggers Callbacks">
    **skuInventoryChange** callbacks are sent for each affected SKU
  </Step>
  <Step title="Creates Audit Trail">
    All shipment changes are logged for tracking and reconciliation
  </Step>
</Steps>

## Receiving Events

After creating a shipment, you'll receive **inboundShipmentEvent** callbacks as the shipment progresses:

- **onDock**: Shipment arrives at facility
- **receivingStarted**: Receiving process begins
- **receivingComplete**: All items processed (includes line item details and discrepancies)
