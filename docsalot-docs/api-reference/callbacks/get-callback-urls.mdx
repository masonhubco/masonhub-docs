---
title: 'Get Callback URLs'
description: 'Retrieve registered callback URLs filtered by message type'
api: 'GET /callbacks'
---

Retrieve all registered webhook callback URLs, optionally filtered by message type. Use this endpoint to view and audit your current webhook subscriptions.

<Note>
Filter by specific message types to reduce response size, or use `message_type=all` to retrieve all callbacks at once.
</Note>

## Query Parameters

<ParamField query="message_type" type="string" default="all">
  Filter callbacks by message type. Use `all` to retrieve all callbacks regardless of type.
</ParamField>

<RequestExample>
```bash cURL
curl -X GET "https://app.masonhub.co/{account}/api/v1/callbacks?message_type=skuInventoryChange" \
  -H "Authorization: Bearer your_jwt_token"
```

```python Python
import requests

url = "https://app.masonhub.co/{account}/api/v1/callbacks"
headers = {
    "Authorization": "Bearer your_jwt_token"
}
params = {
    "message_type": "skuInventoryChange"
}

response = requests.get(url, headers=headers, params=params)
callbacks = response.json()
```

```javascript JavaScript
const response = await fetch('https://app.masonhub.co/{account}/api/v1/callbacks?message_type=skuInventoryChange', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer your_jwt_token'
  }
});

const callbacks = await response.json();
```
</RequestExample>

## Response Fields

Each callback object in the response includes:

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique callback UUID for future operations |
| `url` | string | HTTPS webhook endpoint URL |
| `message_type` | string | Event type this callback is subscribed to |
| `api_version` | string | API version for callback payloads |
| `token` | string | Encrypted verification token (if configured) |
| `created_at` | string | Registration timestamp in RFC3339 format |

## Callback Payload Format

When events occur, MasonHub sends POST requests to registered URLs with this structure:

```json
{
  "callback_url": "https://client.com/api/webhook",
  "message_type": "eventType",
  "message_id": "unique-event-id",
  "api_version": "1.0",
  "data": [
    {
      // Event-specific data
    }
  ]
}
```

## Use Cases

<Tabs>
  <Tab title="Audit Callbacks">
    Periodically fetch all callbacks to ensure your webhook configuration is up to date:

    ```bash
    GET /callbacks?message_type=all
    ```

    Review the list and remove any callbacks pointing to decommissioned endpoints.
  </Tab>

  <Tab title="Verify Registration">
    After registering a new callback, verify it was created successfully:

    ```bash
    GET /callbacks?message_type=orderEvent
    ```

    Check that your URL appears in the results with the correct configuration.
  </Tab>

  <Tab title="Get Callback IDs">
    Retrieve callback IDs for deletion operations:

    ```python
    # Get all callbacks for a message type
    callbacks = get_callbacks("orderEvent")

    # Find specific callback by URL
    callback_to_delete = next(
        c for c in callbacks
        if c['url'] == 'https://old-domain.com/webhook'
    )

    # Use the ID for deletion
    delete_callback(callback_to_delete['id'])
    ```
  </Tab>
</Tabs>
