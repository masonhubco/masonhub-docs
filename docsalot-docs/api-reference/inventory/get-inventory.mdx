The Inventory API provides real-time inventory tracking and snapshot capabilities across all distribution centers. It supports various inventory statuses, detailed location breakdowns, and automatic inventory change notifications.

<Note>
Use callbacks for real-time inventory updates rather than frequent polling to optimize performance and stay current with inventory changes.
</Note>

Retrieve real-time inventory levels for specified SKUs across all locations and statuses. This endpoint provides current inventory counts with detailed breakdowns by location and status.


## Query Parameters

<ParamField query="cid" type="string[]" required>
  Customer SKU identifiers. Supports batching 1-30 identifiers per request.
</ParamField>

<ParamField query="limit" type="integer" default="30">
  Number of results to return. Range: 1-100.
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Pagination offset for retrieving subsequent pages of results.
</ParamField>

<ParamField query="include_ledger" type="boolean" default="true">
  Include detailed inventory breakdown by location and status.
</ParamField>

<RequestExample>
```bash cURL
curl -X GET "https://app.masonhub.co/{account}/api/v1/sku_snapshots?cid=pants4523,shirts872340&limit=50" \
  -H "Authorization: Bearer your_jwt_token"
```

```python Python
import requests

url = "https://app.masonhub.co/{account}/api/v1/sku_snapshots"
headers = {
    "Authorization": "Bearer your_jwt_token"
}
params = {
    "cid": "pants4523,shirts872340",
    "limit": 50
}

response = requests.get(url, headers=headers, params=params)
data = response.json()
```


## Inventory Status Types

The response includes inventory broken down by status:

<CardGroup cols={2}>
  <Card title="Sellable Inventory" icon="check-circle">
    - **available**: Ready for sale and allocation
    - **expected**: Incoming from inbound shipments
    - **received**: Recently received, being processed
  </Card>
  <Card title="Quality Control" icon="search">
    - **quality-control**: Under inspection
    - **refurbishing**: Being repaired/refurbished
    - **under-investigation**: Status being determined
  </Card>
  <Card title="Processing States" icon="cog">
    - **in-receiving**: Currently being processed
    - **allocated**: Reserved for orders
    - **picked**: Retrieved for shipment
  </Card>
  <Card title="Unavailable" icon="xmark">
    - **damaged**: Requires disposition
    - **lost**: Cannot be located
    - **expired**: Past expiration date
  </Card>
</CardGroup>

## Inventory Calculations

### Total Units
Sum of all inventory across all statuses and locations for a SKU.

### Total Available to Sell
Inventory that can be allocated to new orders:

```
Available to Sell = Available + Expected + Received
```

### Kit Inventory
For kit SKUs, inventory calculations automatically consider:
- Pre-built kit quantities 
- Component availability for kit-to-ship
- Mixed inventory scenarios

## Batch Queries

Query multiple SKUs in a single request:

```bash
GET /sku_snapshots?cid=sku001,sku002,sku003&limit=50
```

<Warning>
Maximum 30 customer identifiers per request. Exceeding this limit will result in a 400 error.
</Warning>
