Create one or more SKUs in the MasonHub catalog system. SKUs are the foundation of your product catalog and contain all the information needed for storage, fulfillment, and shipping.

<Note>
SKUs can be created individually or in batches. For large catalog uploads, consider batching up to 100 SKUs per request.
</Note>

## Request Body

The request body accepts an array of SKU objects. Each SKU object supports the following fields:

### Required Fields

<ParamField body="customer_identifier" type="string" required>
  Unique customer SKU identifier. This is your primary reference for the SKU.
</ParamField>

<ParamField body="customer_bar_code" type="string" required>
  Primary customer-assigned barcode for warehouse scanning.
</ParamField>

<ParamField body="vendor_style" type="string" required>
  Vendor style code. Must be 1-40 characters.
</ParamField>

<ParamField body="brand_name" type="string" required>
  Brand name for the product.
</ParamField>

<ParamField body="product_name" type="string" required>
  Product name/title.
</ParamField>

<ParamField body="product_category" type="string" required>
  Product category classification (e.g., "Apparel", "Electronics").
</ParamField>

<ParamField body="dim_weight" type="number" required>
  Weight in pounds.
</ParamField>

<ParamField body="dim_length" type="number" required>
  Length in inches.
</ParamField>

<ParamField body="dim_width" type="number" required>
  Width in inches.
</ParamField>

<ParamField body="dim_height" type="number" required>
  Height in inches.
</ParamField>

### Optional Fields

<ParamField body="vendor_bar_code" type="string">
  Vendor's original barcode.
</ParamField>

<ParamField body="bar_codes" type="string[]">
  Additional barcodes for the SKU. Useful for multiple UPC codes or regional variations.
</ParamField>

<ParamField body="product_sub_category" type="string">
  Product subcategory for finer classification.
</ParamField>

<ParamField body="product_description" type="string">
  Detailed product description.
</ParamField>

<ParamField body="dim_volume" type="number">
  Volume in cubic inches. Calculated automatically if not provided.
</ParamField>

<ParamField body="unit_cost" type="number">
  Unit cost for the SKU.
</ParamField>

<ParamField body="price" type="number">
  Retail price for the SKU.
</ParamField>

<ParamField body="pick_style" type="string">
  Pick strategy for inventory allocation. Options:
  - `FIFO` - First In, First Out (default)
  - `FEFO` - First Expired, First Out
  - `LIFO` - Last In, First Out
</ParamField>

<ParamField body="special_instructions" type="string">
  Special handling instructions for warehouse operations.
</ParamField>

<ParamField body="is_kit" type="boolean">
  Whether this SKU is a kit composed of multiple child SKUs.
</ParamField>

<ParamField body="kit_skus" type="array">
  Array of component SKUs for kits. Each item requires `customer_identifier` and `quantity`.
</ParamField>

## Kit SKUs

MasonHub supports kit SKUs - parent SKUs composed of multiple child SKUs. There are two types:

<Tabs>
  <Tab title="Pre-built Kits">
    **Characteristics:**
    - Already assembled at the warehouse
    - Child units not available for individual sale
    - Faster processing and shipping
    - Single inventory tracking unit
    
    **Best for:** Gift sets, promotional bundles, pre-configured packages
  </Tab>
  <Tab title="Kit-to-Ship">
    **Characteristics:**
    - Assembled when orders arrive
    - Child units remain available individually
    - More flexible inventory management
    - Component-level tracking
    
    **Best for:** Custom configurations, build-to-order products, flexible bundles
  </Tab>
</Tabs>

### Kit SKU Example

```json
{
  "customer_identifier": "kit00001",
  "customer_bar_code": "1234567890",
  "vendor_style": "KIT-outfit",
  "brand_name": "Freddie's",

  "product_category": "Apparel",
  "product_description": "Complete outfit kit",
  "is_kit": true,
  "kit_skus": [
    {
      "customer_identifier": "shirt-001",
      "quantity": 1
    },
    {
      "customer_identifier": "pants-001",
      "quantity": 1
    }
  ],
  "dim_weight": 1.25,
  "dim_length": 12,
  "dim_width": 10,
  "dim_height": 3
}
```

<Warning>
Child SKUs must exist in the catalog before creating a kit SKU that references them.
</Warning>

## Barcode Management

SKUs support multiple barcode associations:

| Field | Description |
|-------|-------------|
| `customer_bar_code` | Primary customer barcode (required, must be unique) |
| `vendor_bar_code` | Vendor's barcode (optional, must be unique) |
| `bar_codes` | Array of additional barcodes (all must be unique) |

<Warning>
All barcodes must be globally unique across the system. The API will return a 409 error if duplicate barcodes are detected.
</Warning>
