Master order management with MasonHub's comprehensive system. This guide covers everything from order creation to fulfillment and tracking.

## Order Lifecycle

<Steps>
  <Step title="Order Creation">
    Orders enter the system via integration, API, or manual entry
  </Step>
  <Step title="Validation">
    System validates inventory, address, and payment
  </Step>
  <Step title="Processing">
    Order is allocated to warehouse and picked
  </Step>
  <Step title="Fulfillment">
    Order is packed and shipped
  </Step>
  <Step title="Delivery">
    Tracking provided until successful delivery
  </Step>
</Steps>

## Creating Orders

### Order Creation Methods

<Tabs>
  <Tab title="Automated Import">
    ### Integration-Based Orders

    Orders flow automatically from your connected platforms:

    **Shopify Orders:**
    - Auto-import when order is paid
    - Real-time inventory sync
    - Automatic status updates

    **EDI Orders:**
    - Batch processing
    - Compliance validation
    - Automated acknowledgments

    **API Orders:**
    - POST to `/orders` endpoint
    - Real-time validation
    - Immediate confirmation

    [API Documentation →](https://support.masonhub.co/hc/en-us/articles/360052470011-Create-Edit-Cancel-Orders-MasonHub-API)
  </Tab>

  <Tab title="Manual Entry">
    ### Create Orders Manually

    Perfect for samples, replacements, or one-off orders:

    1. Navigate to **Orders** → **Create New**
    2. Select order type:
       - B2C (Direct to Consumer)
       - B2B (Wholesale)
       - Sample
       - Replacement
    3. Enter customer information
    4. Add products and quantities
    5. Select shipping method
    6. Review and submit

    <Tip>
      Use manual orders for testing new workflows before automation.
    </Tip>
  </Tab>

  <Tab title="Bulk Upload">
    ### CSV Import

    Import multiple orders at once:

    1. Download CSV template
    2. Fill in order details
    3. Upload via **Orders** → **Bulk Import**
    4. Review validation results
    5. Confirm import

    **Required fields:**
    - Order number
    - Customer name
    - Shipping address
    - SKU and quantity
    - Shipping method
  </Tab>
</Tabs>

### Editing Orders

<Warning>
  Orders can only be edited before warehouse processing begins. Once picking starts, cancellation and re-creation is required.
</Warning>

**Editable fields:**
- Shipping address
- Shipping method
- Order priority
- Special instructions
- Hold status

**To edit an order:**
1. Find order in **Orders** list
2. Click **Edit** (if available)
3. Make necessary changes
4. Save and confirm

### Cancelling Orders

<Tabs>
  <Tab title="Single Cancellation">
    **Cancel individual orders:**

    1. Locate the order
    2. Click **Cancel Order**
    3. Select cancellation reason
    4. Confirm cancellation

    **Timing:**
    - Before picking: Immediate cancellation
    - During fulfillment: Attempt to stop
    - After shipping: Return process required
  </Tab>

  <Tab title="Bulk Cancellation">
    **Cancel multiple orders:**

    1. Go to **Orders** → **Bulk Actions**
    2. Select orders to cancel
    3. Choose **Cancel Selected**
    4. Provide reason
    5. Confirm action

    <Note>
      Bulk cancellations are logged for audit purposes.
    </Note>
  </Tab>

  <Tab title="API Cancellation">
    **Programmatic cancellation:**

    ```json
    DELETE /api/orders/{orderId}
    {
      "reason": "customer_request",
      "notify_customer": true
    }
    ```

    [API Reference →](/api-reference/orders/cancel-orders)
  </Tab>
</Tabs>

## Reporting & Analytics

### Order Reports

Access comprehensive order data:

- **Daily Order Summary** - Volume, status, performance
- **Fulfillment Metrics** - Processing times, accuracy
- **Shipping Analysis** - Carrier performance, costs
- **Exception Reports** - Holds, cancellations, returns

**To generate reports:**
1. Navigate to **Reports** → **Orders**
2. Select report type
3. Choose date range
4. Apply filters
5. Export (CSV, Excel, PDF)
