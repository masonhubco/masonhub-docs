## Key Documentation Sections

### Core API Endpoints

Review these essential endpoint categories:

<Accordion title="Orders API">
  - **Create Orders**: Submit new orders to MasonHub
  - **Get Orders**: Retrieve order status and details
  - **Update Orders**: Modify existing orders
  - **Order Events**: Webhook notifications for order changes
</Accordion>

<Accordion title="SKUs API">
  - **Create SKUs**: Add products to your catalog
  - **Update SKUs**: Modify product information
  - **Get SKUs**: Retrieve product details
  - **Bulk Operations**: Handle large product catalogs efficiently
</Accordion>

<Accordion title="Inbound Shipments API">
  - **Create Inbound Shipments**: Schedule inventory receipts
  - **Track Receipts**: Monitor receiving progress
  - **Shipment Events**: Get notified of receipt status changes
</Accordion>

<Accordion title="Inventory API">
  - **Inventory Snapshots**: Real-time inventory levels
  - **Availability Checks**: Verify stock before order creation
  - **Allocation Management**: Understand committed vs. available inventory
</Accordion>

## Integration Patterns

### Authentication

All API calls require authentication using your bearer token:

```bash
curl -H "Authorization: Bearer YOUR_API_TOKEN" \
     -H "Content-Type: application/json" \
     https://api.masonhub.co/api/v1/orders
```

### Rate Limiting

<Note>
The MasonHub API implements rate limiting to ensure system stability. Review the rate limit headers in API responses and implement appropriate retry logic.
</Note>

### Error Handling

Understand the API's error response format:

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid ship method mapping",
    "details": {
      "field": "shipping_service_provider",
      "value": "invalid_provider"
    }
  }
}
```

## Environment Differences

### Sandbox Environment
- **Base URL**: `https://sandbox-api.masonhub.co`
- **Purpose**: Testing and development
- **Data**: Test data that can be safely modified

### Production Environment
- **Base URL**: `https://api.masonhub.co`
- **Purpose**: Live operations
- **Data**: Real customer and order data

<Warning>
Always develop and test in the sandbox environment before deploying to production.
</Warning>
