Create return merchandise authorizations (RMA) for customer returns. The system supports automatic DHL return label generation and various return types.

<Note>
MasonHub automatically generates DHL return labels when `generate_return_label` is true (default) and complete customer address is provided.
</Note>

<Warning>
Return label URLs require API authentication and contain customer personal information. Treat them as sensitive data.
</Warning>

## Request Body Schema

Send an array of RMA objects. Multiple returns can be created in a single request.

### Required Fields

<ParamField body="customer_identifier" type="string" required>
  Unique RMA identifier in your system. Used to reference this return in future operations.
</ParamField>

<ParamField body="return_type" type="string" required>
  Type of return. Options:
  - `rma_submitted_by_customer` - Customer-initiated return
  - `quality_control` - Quality control return
  - `damaged_in_transit` - Damaged during shipping
  - `wrong_item_shipped` - Incorrect item shipped
</ParamField>

<ParamField body="line_items" type="array" required>
  Array of items being returned. Each item must include:
  - `customer_sku_id` (string, required): Your SKU identifier
  - `quantity` (integer, required): Quantity being returned
  - `return_reason_code` (string, required): Reason for return
  - `return_inventory_status` (string, required): Target inventory status
  - `customer_order_id` (string, optional): Original order reference
  - `return_notes` (string, optional): Additional notes
</ParamField>

### Optional Fields

<ParamField body="manifest_id" type="string">
  Manifest identifier for grouping related returns.
</ParamField>

<ParamField body="generate_return_label" type="boolean" default="true">
  Whether to generate a DHL return label. Requires complete customer address when true.
</ParamField>

<ParamField body="customer_instructions" type="string">
  Special handling instructions for warehouse team (e.g., "Dry clean before restocking").
</ParamField>

### Customer Address Fields

<ParamField body="customer_address_name" type="string">
  Customer name. Required if `generate_return_label` is true.
</ParamField>

<ParamField body="customer_address_street_line_one" type="string">
  Street address line 1. Required if `generate_return_label` is true.
</ParamField>

<ParamField body="customer_address_street_line_two" type="string">
  Street address line 2 (apartment, suite, etc.). Optional.
</ParamField>

<ParamField body="customer_address_city" type="string">
  City. Required if `generate_return_label` is true.
</ParamField>

<ParamField body="customer_address_state" type="string">
  State or province code (e.g., "NY", "CA"). Required if `generate_return_label` is true.
</ParamField>

<ParamField body="customer_address_postal_code" type="string">
  Postal/ZIP code. Required if `generate_return_label` is true.
</ParamField>

<ParamField body="customer_address_country_code" type="string">
  Country code (e.g., "US", "CA"). Required if `generate_return_label` is true.
</ParamField>

## Return Reason Codes

<ParamField body="return_reason_code" type="string">
  Standardized reason codes for analytics:
  - `tooBig` - Item was too large
  - `tooSmall` - Item was too small
  - `damaged` - Customer received damaged item
  - `wrongItem` - Customer received wrong item
  - `other` - Reason not captured in standard codes
</ParamField>

## Return Inventory Statuses

<ParamField body="return_inventory_status" type="string">
  Target inventory status for returned items:
  - `available` - Ready for resale
  - `quality-control` - Requires inspection
  - `damaged` - Damaged items
  - `refurbishing` - Needs refurbishment
  - `under-investigation` - Under review
</ParamField>

## Return Label Generation

When creating an RMA with return label generation:

<Steps>
  <Step title="Validate Address">
    System validates that all required address fields are provided
  </Step>
  <Step title="Generate Label">
    DHL return label is automatically generated
  </Step>
  <Step title="Return URLs">
    Response includes `return_label_url` and `return_tracking_number`
  </Step>
  <Step title="Email Label">
    Optionally email the label directly to the customer
  </Step>
</Steps>

## Order Dependencies

<Info>
RMAs can be created with or without order references. This flexibility supports:
- Historical returns during system go-live
- Returns without order tracking
- Cross-order returns (single RMA referencing multiple orders)
</Info>

When `customer_order_id` is provided, the system validates:
- Order exists in the system
- SKU was on the referenced order
- Quantity doesn't exceed original order quantity

## Cross-Order Returns

A single RMA can reference multiple orders at the line item level:

```json

  "customer_identifier": "rma123",
```

## Callback Events

After creating an RMA, you'll receive **rmaEvent** callbacks as the return progresses:

- **tendered**: Package picked up by carrier
- **inTransit**: Package in transit to facility
- **delivered**: Package arrived at facility
- **received**: Items processed into inventory (includes line item details)
