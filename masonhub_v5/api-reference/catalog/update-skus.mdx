---
title: 'Update SKUs'
description: 'Update existing SKUs on the Item Master catalog'
api: 'PUT /skus'
---

Update one or more existing SKUs in the MasonHub catalog. This endpoint uses the `customer_identifier` to locate and update SKUs.

<Warning>
**Full Replacement**: Always send the entire SKU object in the request body. Whatever you send will completely replace the existing SKU data in MasonHub's systems. We have elected not to use the PATCH verb to avoid complicating your integrations with us.
</Warning>

## Request Body

The request body accepts an array of SKU objects with the same schema as the Create SKUs endpoint. The `customer_identifier` field is used to match existing SKUs.

### Required Fields

<ParamField body="customer_identifier" type="string" required>
  Unique customer SKU identifier. Used to locate the existing SKU to update.
</ParamField>

<ParamField body="customer_bar_code" type="string" required>
  Primary customer-assigned barcode for warehouse scanning.
</ParamField>

<ParamField body="vendor_style" type="string" required>
  Vendor style code. Must be 1-40 characters.
</ParamField>

<ParamField body="brand_name" type="string" required>
  Brand name for the product.
</ParamField>

<ParamField body="product_name" type="string" required>
  Product name/title.
</ParamField>

<ParamField body="product_category" type="string" required>
  Product category classification.
</ParamField>

<ParamField body="dim_weight" type="number" required>
  Weight in pounds.
</ParamField>

<ParamField body="dim_length" type="number" required>
  Length in inches.
</ParamField>

<ParamField body="dim_width" type="number" required>
  Width in inches.
</ParamField>

<ParamField body="dim_height" type="number" required>
  Height in inches.
</ParamField>

### Optional Fields

<ParamField body="vendor_bar_code" type="string">
  Vendor's original barcode.
</ParamField>

<ParamField body="bar_codes" type="string[]">
  Additional barcodes for the SKU.
</ParamField>

<ParamField body="product_sub_category" type="string">
  Product subcategory for finer classification.
</ParamField>

<ParamField body="product_description" type="string">
  Detailed product description.
</ParamField>

<ParamField body="dim_volume" type="number">
  Volume in cubic inches.
</ParamField>

<ParamField body="unit_cost" type="number">
  Unit cost for the SKU.
</ParamField>

<ParamField body="price" type="number">
  Retail price for the SKU.
</ParamField>

<ParamField body="pick_style" type="string">
  Pick strategy: `FIFO`, `FEFO`, or `LIFO`.
</ParamField>

<ParamField body="special_instructions" type="string">
  Special handling instructions for warehouse operations.
</ParamField>

<ParamField body="is_kit" type="boolean">
  Whether this SKU is a kit.
</ParamField>

<ParamField body="kit_skus" type="array">
  Array of component SKUs for kits.
</ParamField>

## Update Best Practices

<Steps>
  <Step title="Fetch Current SKU">
    First retrieve the current SKU data using `GET /skus?cid=your_sku_id`
  </Step>
  <Step title="Modify Fields">
    Update only the fields you want to change in your local copy
  </Step>
  <Step title="Send Complete Object">
    Send the entire modified SKU object in the PUT request
  </Step>
  <Step title="Verify Response">
    Check the response to confirm the update was successful
  </Step>
</Steps>

<Note>
**Why Full Replacement?** This approach ensures data consistency and prevents partial updates that could leave SKUs in an inconsistent state. It also simplifies error handling and rollback scenarios.
</Note>

## Common Update Scenarios

<Accordion title="Price Updates">
  When updating prices, remember to include all other fields:
  
  ```json
  {
    "customer_identifier": "pants4523",
    "customer_bar_code": "2345687902",
    // ... all other existing fields ...
    "unit_cost": 90,   // New cost
    "price": 200       // New retail price
  }
  ```
</Accordion>

<Accordion title="Adding Barcodes">
  To add new barcodes, include the complete barcode array:
  
  ```json
  {
    "customer_identifier": "pants4523",
    // ... all other existing fields ...
    "bar_codes": [
      "85962198623",      // Existing
      "3459189485324",    // Existing
      "NEW123456789"      // New barcode
    ]
  }
  ```
</Accordion>

<Accordion title="Changing Pick Strategy">
  Update the pick style for inventory allocation:
  
  ```json
  {
    "customer_identifier": "pants4523",
    // ... all other existing fields ...
    "pick_style": "FEFO"  // Changed from FIFO to FEFO
  }
  ```
</Accordion>

## Batch Updates

You can update multiple SKUs in a single request:

```json
[
  {
    "customer_identifier": "pants4523",
    // ... complete SKU object ...
  },
  {
    "customer_identifier": "shirts450293",
    // ... complete SKU object ...
  }
]
```

The response will include the status of each individual update in the `results` array.
