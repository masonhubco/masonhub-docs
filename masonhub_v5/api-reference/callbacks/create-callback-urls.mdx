Register webhook URLs to receive real-time event notifications from MasonHub. Subscribe to multiple event types by creating separate callbacks for each.

<Warning>
All callback URLs must use **HTTPS**. HTTP URLs will be rejected for security reasons.
</Warning>

## Request Body

Send an array of callback objects to register. Each callback object supports:

<ParamField body="url" type="string" required>
  HTTPS endpoint URL that will receive webhook notifications. Must be publicly accessible.
</ParamField>

<ParamField body="message_type" type="string" required>
  Event type to subscribe to. See available message types below.
</ParamField>

<ParamField body="api_version" type="string" default="1.0">
  API version for callback payloads. Defaults to "1.0".
</ParamField>

<ParamField body="token" type="string">
  Optional verification token to include in callback requests for authentication. Recommended for production.
</ParamField>

## Available Message Types

<AccordionGroup>
  <Accordion title="skuInventoryChange">
    **Triggered when**: Inventory levels change for any SKU (received, allocated, shipped)

    **Use for**: Real-time inventory synchronization, low stock alerts, reorder automation
  </Accordion>

  <Accordion title="orderEvent">
    **Triggered when**: Order status changes (packed, fulfilled, delivered, canceled)

    **Use for**: Order tracking, customer notifications, shipping updates
  </Accordion>

  <Accordion title="orderUpdateResolution">
    **Triggered when**: Order update requests complete (success or failure)

    **Use for**: Confirming order modifications, handling update failures
  </Accordion>

  <Accordion title="orderCancelResolution">
    **Triggered when**: Order cancellation requests complete (success or failure)

    **Use for**: Confirming cancellations, processing refunds
  </Accordion>

  <Accordion title="inboundShipmentEvent">
    **Triggered when**: Inbound shipment status changes (created, received, discrepancies)

    **Use for**: Tracking inventory receipts, handling discrepancies
  </Accordion>

  <Accordion title="rmaEvent">
    **Triggered when**: Return merchandise authorization status changes

    **Use for**: Return processing, customer service workflows
  </Accordion>

  <Accordion title="snapshotReady">
    **Triggered when**: Inventory snapshot generation completes

    **Use for**: Scheduled inventory reports, data warehouse updates
  </Accordion>
</AccordionGroup>

## Security

### HTTPS Requirement

All callback URLs must use HTTPS for secure transmission:

<CodeGroup>
```json ✅ Valid - HTTPS
{
  "url": "https://api.clienturl.com/webhook"
}
```

```json ❌ Invalid - HTTP
{
  "url": "http://api.clienturl.com/webhook"
}
```
</CodeGroup>

### Verification Tokens

Include tokens for authenticating callback requests:

```json
{
  "url": "https://api.clienturl.com/webhook",
  "message_type": "orderEvent",
  "token": "your_encrypted_verification_token"
}
```

The token will be included in all callback requests. Your endpoint should verify the token matches before processing events.

## Webhook Endpoint Requirements

Your webhook endpoints must meet these requirements:

<Steps>
  <Step title="Accept POST Requests">
    All callbacks use the POST method with JSON payloads
  </Step>
  <Step title="Return HTTP 200">
    Respond with 200 status code to indicate successful processing
  </Step>
  <Step title="Respond Quickly">
    Return response within 30 seconds to prevent timeouts
  </Step>
  <Step title="Handle Duplicates">
    Implement idempotency using `message_id` to handle at-least-once delivery
  </Step>
  <Step title="Validate Authenticity">
    Verify token when configured to ensure requests are from MasonHub
  </Step>
</Steps>

## Delivery Guarantees

<CardGroup cols={2}>
  <Card title="At-Least-Once Delivery" icon="repeat">
    Events may be delivered multiple times. Implement idempotency using `message_id`.
  </Card>
  <Card title="Retry Logic" icon="arrows-rotate">
    Failed deliveries are retried with exponential backoff up to 24 hours.
  </Card>
  <Card title="Timeout Handling" icon="clock">
    Requests timeout after 30 seconds. Respond quickly to avoid retries.
  </Card>
  <Card title="Error Logging" icon="file-lines">
    Delivery failures are logged and available on request for troubleshooting.
  </Card>
</CardGroup>
