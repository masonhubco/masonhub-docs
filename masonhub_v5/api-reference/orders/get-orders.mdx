title: 'Get Orders'

---

Retrieve orders from the MasonHub fulfillment system. This endpoint supports both detailed and summary views, with flexible filtering options for efficient data retrieval.

<Note>
Orders are returned in descending order by creation date. Use pagination parameters for large result sets.
</Note>

## Query Parameters

Configure your request with these query parameters:

### Primary Filters

<ParamField query="cid" type="string">
  Filter by customer identifier. Returns specific order by your reference ID.
</ParamField>

<ParamField query="id" type="string">
  Filter by MasonHub system ID (UUID). Use for precise order retrieval.
</ParamField>

<ParamField query="list_type" type="string" default="summary">
  Response detail level:
  - `summary` - Basic order information (default)
  - `detail` - Complete order data including all fields
</ParamField>

<ParamField query="limit" type="integer" default="100">
  Maximum number of records to return (1-1000).
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Number of records to skip for pagination.
</ParamField>

### Date Filters

<ParamField query="created_after" type="string">
  Filter orders created after this timestamp (RFC3339 format).
</ParamField>

<ParamField query="created_before" type="string">
  Filter orders created before this timestamp (RFC3339 format).
</ParamField>

<ParamField query="updated_after" type="string">
  Filter orders updated after this timestamp (RFC3339 format).
</ParamField>

<ParamField query="updated_before" type="string">
  Filter orders updated before this timestamp (RFC3339 format).
</ParamField>

### Status Filters

<ParamField query="status" type="string">
  Filter by order status. Options:
  - `pending` - Awaiting processing
  - `processing` - Being picked/packed
  - `fulfilled` - Shipped
  - `delivered` - Confirmed delivery
  - `canceled` - Order canceled
  - `on_hold` - Temporarily paused
</ParamField>

<ParamField query="has_shipments" type="boolean">
  Filter orders by shipment existence:
  - `true` - Only orders with shipments
  - `false` - Only orders without shipments
</ParamField>

```bash cURL
# Get specific order by customer ID

  -H "Authorization: Bearer your_jwt_token"

curl -X GET "https://app.masonhub.co/{account}/api/v1/orders?list_type=detail&limit=50" \
  -H "Authorization: Bearer your_jwt_token"

# Get orders created in date range
curl -X GET "https://app.masonhub.co/{account}/api/v1/orders?created_after=2024-01-01T00:00:00Z&created_before=2024-01-31T23:59:59Z" \
  -H "Authorization: Bearer your_jwt_token"
```

```python Python
import requests

# Get specific order
url = "https://app.masonhub.co/{account}/api/v1/orders"
headers = {"Authorization": "Bearer your_jwt_token"}
params = {"cid": "129374", "list_type": "detail"}

response = requests.get(url, headers=headers, params=params)
order = response.json()

# Get paginated orders
params = {
    "limit": 100,
    "offset": 0,
    "list_type": "summary",
    "status": "processing"
}

response = requests.get(url, headers=headers, params=params)
orders = response.json()
```

## Pagination

Handle large result sets using pagination:

<Steps>
  <Step title="Initial Request">
    Start with `offset=0` and your desired `limit` (max 1000)
  </Step>
  <Step title="Check Total Count">
    Response includes `total_count` field showing total matching orders
  </Step>
  <Step title="Next Page">
    Increment `offset` by `limit` value for next page
  </Step>
  <Step title="Continue">
    Repeat until `offset + limit >= total_count`
  </Step>
</Steps>

Example pagination flow:
```python
def get_all_orders(headers):
    all_orders = []
    offset = 0
    limit = 100

    while True:
        params = {"limit": limit, "offset": offset}
        response = requests.get(url, headers=headers, params=params)
        data = response.json()

        all_orders.extend(data["orders"])

        if offset + limit >= data["total_count"]:
            break

        offset += limit

    return all_orders
```

## Order Status Lifecycle

Orders progress through these statuses:

<Steps>
  <Step title="pending">
    Order received and awaiting processing
  </Step>
  <Step title="processing">
    Order being picked and packed in warehouse
  </Step>
  <Step title="fulfilled">
    Order shipped with tracking information available
  </Step>
  <Step title="delivered">
    Carrier confirmed delivery to recipient
  </Step>
</Steps>

Additional statuses:
- `canceled` - Order canceled before fulfillment
- `on_hold` - Order temporarily paused (inventory, payment, or manual hold)
- `returned` - Order returned after delivery

## Best Practices

<CardGroup cols={2}>
  <Card title="Use Specific Filters" icon="filter">
    Apply date ranges and status filters to reduce response size and improve performance.
  </Card>
  <Card title="Choose Right Detail Level" icon="list">
    Use `summary` for lists and dashboards, `detail` only when full data needed.
  </Card>
  <Card title="Implement Pagination" icon="forward">
    Always paginate when retrieving multiple orders to avoid timeouts.
  </Card>
  <Card title="Cache When Possible" icon="database">
    Orders in `fulfilled` or `delivered` status rarely change - cache these locally.
  </Card>
</CardGroup>
