---
title: 'Delete Secret Phrase'
description: 'Remove secret phrases to invalidate associated bearer tokens'
api: 'POST /delete_secrets'
---

Remove secret phrases from your account, permanently invalidating all bearer tokens generated with that phrase. This is useful for token rotation and security management.

<Warning>
Deleting a secret phrase invalidates **all tokens** generated with that phrase. This action cannot be undone.
</Warning>

<Note>
The secret phrase must match **exactly** as it was entered during token creation, including capitalization, spacing, and punctuation.
</Note>

## Request Body

<ParamField body="secret_phrase" type="string" required>
  The exact secret phrase used to generate the token(s) you want to invalidate. Must match character-for-character with the original phrase.
</ParamField>

<RequestExample>
```bash cURL
curl -X POST "https://app.masonhub.co/{account}/api/v1/delete_secrets" \
  -H "Content-Type: application/json" \
  -d '{
    "secret_phrase": "Enter secret phrase exactly as entered during creation process"
  }'
```

```python Python
import requests

url = "https://app.masonhub.co/{account}/api/v1/delete_secrets"
headers = {
    "Content-Type": "application/json"
}
payload = {
    "secret_phrase": "Enter secret phrase exactly as entered during creation process"
}

response = requests.post(url, headers=headers, json=payload)
result = response.json()
print(f"Secret phrase deleted: {result}")
```

## When to Delete Secret Phrases

<AccordionGroup>
  <Accordion title="Token Rotation">
    Part of regular security maintenance - delete old phrases after rotating to new tokens.
  </Accordion>

  <Accordion title="Security Breach">
    If a token is compromised, immediately delete the secret phrase to invalidate all associated tokens.
  </Accordion>

  <Accordion title="Employee Offboarding">
    Remove access by deleting secret phrases when team members leave or change roles.
  </Accordion>

  <Accordion title="Environment Cleanup">
    Clean up test or temporary tokens that are no longer needed.
  </Accordion>

  <Accordion title="Integration Decommission">
    Remove tokens when decommissioning integrations or applications.
  </Accordion>
</AccordionGroup>

## Token Rotation Workflow

Safely rotate tokens to maintain security:

<Steps>
  <Step title="Generate New Token">
    Create a new bearer token with a different secret phrase using [Generate New Bearer Token](/api-reference/tokens/generate-new-bearer-token)
  </Step>
  <Step title="Update Applications">
    Deploy the new token to all applications and services
  </Step>
  <Step title="Test Thoroughly">
    Verify all integrations work correctly with the new token
  </Step>
  <Step title="Monitor for Issues">
    Watch for authentication errors after deployment
  </Step>
  <Step title="Delete Old Phrase">
    Once confirmed working, delete the old secret phrase to invalidate the previous token
  </Step>
</Steps>

## Important Considerations

<Warning>
**Before Deleting:**
- Ensure you have a new working token in place
- Update all applications using the old token
- Test that the new token works correctly
- Document which systems were updated
</Warning>

### Immediate Effects

When you delete a secret phrase:
- All tokens generated with that phrase are immediately invalidated
- Any API requests using those tokens will return 401 Unauthorized
- The deletion cannot be reversed
- You'll need to generate a new token to regain access

## Emergency Token Revocation

If a token is compromised:

<Steps>
  <Step title="Act Immediately">
    Delete the secret phrase as soon as you detect a security issue
  </Step>
  <Step title="Generate New Token">
    Create a new token with a completely different secret phrase
  </Step>
  <Step title="Review Access Logs">
    Check API access logs for suspicious activity
  </Step>
  <Step title="Update Security">
    Review and strengthen your token storage and handling practices
  </Step>
  <Step title="Notify Stakeholders">
    Inform relevant team members about the security incident
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Secret Phrase Not Found">
    The phrase doesn't match any existing secret. Verify you're using the exact phrase including capitalization and punctuation.
  </Accordion>

  <Accordion title="Applications Still Working">
    Some applications might be caching tokens. Restart services or clear caches if they continue working after deletion.
  </Accordion>

  <Accordion title="Can't Remember Phrase">
    If you've lost the secret phrase, contact support@masonhub.co to manually revoke the associated tokens.
  </Accordion>
</AccordionGroup>

<Info>
For security-related issues or to report compromised tokens, contact **support@masonhub.co** immediately.
</Info>
